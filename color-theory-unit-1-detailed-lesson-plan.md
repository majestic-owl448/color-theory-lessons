# Unit 1 Detailed Lesson Plan: Seeing and Describing Color

## Unit summary
This unit introduces the learner to the basic language of color and teaches them how to describe visible color changes in practical UI terms. The goal is not art-school theory for its own sake. The goal is to help beginner designers look at an interface and explain what color is doing.

This unit should come before technical color formats, accessibility rules, and programming-specific color systems. It establishes the visual vocabulary the learner will use throughout the rest of the app.

## Unit goal
Build the learner's visual vocabulary so they can identify, describe, and intentionally adjust color in digital interfaces.

## Unit outcomes
By the end of Unit 1, the learner should be able to:
- explain what color is doing in a UI at a basic practical level
- distinguish hue, saturation, and lightness/value in plain language
- recognize how contrast affects readability and emphasis
- identify warm and cool color tendencies without over-relying on stereotypes
- explain how color can create visual hierarchy
- recognize basic color relationships and simple harmony patterns
- critique a simple interface based on color clarity and usability

## Unit structure
- **Target lesson count:** 6 lessons
- **Estimated lesson length:** 10 to 15 minutes each
- **Estimated unit duration:** 90 to 120 minutes including milestone
- **Unit milestone:** Read the Interface
- **Milestone badge/title:** Color Observer
- **Milestone route:** `/milestone/milestone-1` (own route)

## Pedagogical approach
This unit should teach through rapid cycles of:
1. short concept introduction
2. visible manipulation of color
3. guided comparison
4. quick decision-making task
5. explanation in the learner's own words

Each lesson should prioritize visual feedback and simple language. Avoid dense theory blocks. When introducing a term, immediately connect it to a visible change on screen.

# Lesson 1: What Color Does in Interface Design

## Lesson purpose
Introduce color as a functional design tool, not just decoration.

## Duration
10 to 12 minutes

## Learning objectives
By the end of this lesson, the learner should be able to:
- explain that color can guide attention, separate sections, and communicate meaning
- identify where color is helping or hurting clarity in a simple interface
- describe color choices in practical terms such as emphasis, grouping, warning, and readability

## Key concepts
- color as communication
- color as emphasis
- color as grouping
- color as status or meaning
- color as decoration versus function

## Key terms
- emphasis
- grouping
- status color
- visual cue
- readability

## Lesson explanation outline
- Designers do not use color only to make things look nice.
- In digital products, color helps users notice what matters first.
- Color can group related items, separate sections, and signal importance.
- Color can communicate status such as success, warning, error, and neutral.
- Bad color choices can make an interface feel confusing, noisy, or hard to read.

## Primary interaction
**Before-and-after interface comparison**
- Show two versions of the same simple UI.
- One version has purposeful color use.
- One version has random or overly loud color use.
- Learner clicks on areas where color improves or harms clarity.

## Guided activity
**Spot the job of the color**
For 4 to 6 highlighted UI examples, the learner chooses what the color is doing:
- drawing attention
- grouping related items
- signaling status
- separating layers
- making text harder to read
- serving no clear purpose

## Practice task
Give the learner a simple dashboard card, form, or landing page hero and ask:
- Which element do you see first?
- Which color is doing the most work?
- Which color feels unnecessary?

## Quiz/check for understanding
- Multiple choice: What is the main job of this accent color?
- Either/or: Is this color functional or decorative?
- Short response: Name one way color can improve a UI.

## Success criteria
The learner can correctly explain at least two practical jobs that color performs in an interface.

## Common mistakes to anticipate
- assuming all bright colors automatically improve design
- treating color as separate from usability
- thinking color only expresses mood and not structure

## Suggested feedback lines
- "Good catch. This color is creating emphasis, not just style."
- "This area uses color decoratively, but it is not improving clarity."
- "Try looking at which item your eye lands on first."

# Lesson 2: Hue, Saturation, and Lightness

## Lesson purpose
Teach the three most important beginner-friendly dimensions used to describe visible color differences.

## Duration
12 to 15 minutes

## Learning objectives
By the end of this lesson, the learner should be able to:
- identify hue, saturation, and lightness/value in practical examples
- predict how moving one slider changes what the learner sees
- describe a color change using the correct term most of the time

## Key concepts
- hue is the family of color
- saturation is intensity or purity
- lightness/value is how light or dark the color appears
- changing one dimension can change usability and hierarchy

## Key terms
- hue
- saturation
- lightness
- value
- muted
- vivid
- tint
- shade

## Lesson explanation outline
- Hue answers the question: which color family is it?
- Saturation answers the question: how intense or muted is it?
- Lightness answers the question: how light or dark is it?
- A designer often adjusts saturation and lightness without changing hue.
- In interfaces, readability and emphasis often depend more on lightness contrast than on hue alone.

## Primary interaction
**Three-slider color lab**
- One target color displayed large.
- Independent sliders for hue, saturation, and lightness.
- Lock two values and let the learner change the third so they can isolate the effect.

## Guided activity
**Name the change**
The app changes a color in one way at a time and the learner chooses:
- hue changed
- saturation changed
- lightness changed

## Practice task
**Reach the target**
Give the learner a target swatch and a starting swatch. Ask them to match it by changing the correct dimension.

## Quiz/check for understanding
- Drag label to the correct slider behavior
- Which change makes the color more muted?
- Which change makes the color darker without changing the color family?

## Success criteria
The learner can correctly identify hue, saturation, and lightness changes in most examples.

## Common mistakes to anticipate
- confusing darker with more saturated
- assuming a hue change happened when only saturation changed
- using "brighter" vaguely when "lighter" or "more saturated" would be more accurate

## Suggested feedback lines
- "The hue stayed similar, but the color became more muted. That is a saturation change."
- "This color feels stronger because saturation increased, not because it got lighter."
- "Try isolating one dimension at a time."

# Lesson 3: Contrast and Readability

## Lesson purpose
Teach the learner to see contrast as a practical usability tool, not just a dramatic visual effect.

## Duration
12 to 15 minutes

## Learning objectives
By the end of this lesson, the learner should be able to:
- explain why contrast matters for readability and focus
- identify low-contrast interface problems visually
- distinguish contrast from simple color difference

## Key concepts
- contrast as difference that affects visibility
- text versus background contrast
- contrast and scanability
- low contrast can make content harder to use even when the palette looks attractive

## Key terms
- contrast
- readability
- legibility
- foreground
- background
- low contrast
- high contrast

## Lesson explanation outline
- Two colors can be different without creating strong usable contrast.
- Contrast matters for text, icons, controls, dividers, and emphasis.
- A beautiful palette can still fail if important information blends into the background.
- Contrast is one of the fastest ways to make hierarchy clearer.

## Primary interaction
**Contrast comparison lab**
- Show multiple text/background combinations.
- Let the learner adjust one color until the text becomes easier or harder to read.
- Include examples where hue changes but readability barely improves.

## Guided activity
**Which one reads faster?**
The learner chooses the more readable version from a pair.

## Practice task
**Repair the card**
A simple UI card has low-contrast labels, buttons, or helper text. The learner adjusts colors until the content becomes easier to read.

## Quiz/check for understanding
- Which pair has stronger text contrast?
- Which problem is the main issue here: hue clash or weak contrast?
- True or false: two very different hues always have strong readable contrast.

## Success criteria
The learner can recognize contrast failures in common UI patterns and improve at least one broken example.

## Common mistakes to anticipate
- confusing strong saturation with strong contrast
- thinking different colors automatically equal readable colors
- focusing on aesthetics before clarity

## Suggested feedback lines
- "These colors are different, but not different enough in visibility."
- "The text blends into the background even though the hues are not the same."
- "Try changing lightness, not just hue."

# Lesson 4: Warm and Cool Colors in Practice

## Lesson purpose
Introduce warm and cool tendencies as a useful but limited design lens.

## Duration
10 to 12 minutes

## Learning objectives
By the end of this lesson, the learner should be able to:
- identify broad warm and cool tendencies in a palette
- explain how warm and cool colors can shift mood and attention
- avoid treating warm/cool labels as hard rules

## Key concepts
- warm versus cool tendencies
- perceived energy and calm
- forward versus background feeling
- context matters more than rigid rules

## Key terms
- warm
- cool
- neutral
- energetic
- calm
- palette mood

## Lesson explanation outline
- Warm colors often feel more active or urgent.
- Cool colors often feel calmer or more distant.
- Neutrals help balance stronger colors.
- Warm/cool effects are useful tendencies, not universal truths.
- Interface context changes how color is perceived.

## Primary interaction
**Palette temperature sorter**
- Learner sorts swatches or mini-palettes into mostly warm, mostly cool, or balanced.
- Then sees an interface where changing palette temperature changes tone.

## Guided activity
**Mood shift challenge**
Take one simple UI and move it toward:
- more energetic
- more calm
- more balanced

## Practice task
Provide three interface goals such as "friendly signup," "serious dashboard," or "urgent alert." Ask the learner which palette direction better fits each goal.

## Quiz/check for understanding
- Which palette feels cooler overall?
- Which change makes the screen feel more urgent?
- Which statement is best: warm colors always mean danger, or warm colors can create energy depending on context?

## Success criteria
The learner can use warm/cool language appropriately and make a basic mood-based palette adjustment.

## Common mistakes to anticipate
- assuming warm always means good and cool always means bad, or the reverse
- treating temperature labels as exact scientific categories in UI practice
- ignoring neutral colors as part of overall palette feel

## Suggested feedback lines
- "Good. This palette feels calmer because the cool tones dominate and the accents are restrained."
- "This version is warmer, but the neutral surfaces still control the overall tone."
- "Think in tendencies, not absolutes."

# Lesson 5: Visual Hierarchy Through Color

## Lesson purpose
Show how designers use color to create order, emphasis, and scanning paths.

## Duration
12 to 15 minutes

## Learning objectives
By the end of this lesson, the learner should be able to:
- explain how color supports visual hierarchy
- identify primary, secondary, and low-emphasis elements in a UI
- adjust color to improve emphasis without making the design noisy

## Key concepts
- hierarchy
- focal point
- emphasis versus support
- primary versus secondary action
- restraint in accent usage

## Key terms
- hierarchy
- focal point
- primary action
- secondary action
- accent color
- de-emphasis

## Lesson explanation outline
- Not all elements should compete equally for attention.
- Color is one of several hierarchy tools, alongside size, spacing, and position.
- A small amount of accent color can be more powerful than color everywhere.
- Surfaces, text, and actions should each have a clear visual role.

## Primary interaction
**Hierarchy tuner**
- Show a cluttered interface with too many equally loud colors.
- Learner adjusts accents, text emphasis, and surface colors to guide attention.

## Guided activity
**Find the first click**
The app asks the learner where a user would likely look first, second, and third. Then it compares that against the intended hierarchy.

## Practice task
**Make one CTA stand out**
Given a screen with three buttons, the learner must make one primary action clearly dominant without destroying balance.

## Quiz/check for understanding
- Which element has the strongest emphasis?
- Why is this hierarchy weak?
- Which revision best supports a single primary action?

## Success criteria
The learner can improve a cluttered interface by reducing noise and strengthening one clear focal point.

## Common mistakes to anticipate
- making all buttons bright
- using multiple competing accent colors
- boosting saturation everywhere instead of designing a hierarchy

## Suggested feedback lines
- "Good hierarchy usually means some elements step back so one element can step forward."
- "Everything cannot be the loudest item on the screen."
- "This accent works better because it is used sparingly."

# Lesson 6: Basic Color Relationships and Harmony

## Lesson purpose
Give the learner a simple practical introduction to color relationships without turning the lesson into advanced theory.

## Duration
12 to 15 minutes

## Learning objectives
By the end of this lesson, the learner should be able to:
- recognize a few simple palette relationship patterns
- choose a basic harmonious direction for a beginner UI palette
- avoid overcomplicating a small interface with too many unrelated colors

## Key concepts
- analogous colors
- complementary contrast
- triadic balance at a basic level
- controlled palette size
- harmony as coordination, not magic

## Key terms
- analogous
- complementary
- triadic
- palette
- harmony
- balance

## Lesson explanation outline
- Some colors feel related because they sit near each other conceptually.
- Some pairings create more tension and contrast.
- Harmony in UI often comes from controlled relationships and restrained use.
- Good digital palettes usually rely on a few roles, not a rainbow.

## Primary interaction
**Palette relationship explorer**
- Interactive wheel or swatch map shows a base hue.
- Learner toggles analogous, complementary, and triadic suggestions.
- Each suggestion is previewed in a simple interface mockup.

## Guided activity
**Choose the better palette**
The learner compares several small UI palettes and explains which feels more coherent and why.

## Practice task
**Build a 3-color starter palette**
The learner creates:
- one dominant color direction
- one supporting color direction
- one accent

Optional neutrals may be added automatically by the system.

## Quiz/check for understanding
- Which pair is closest to complementary?
- Which palette feels more coherent for a beginner UI?
- Why might too many unrelated accents weaken the design?

## Success criteria
The learner can create a small controlled palette and explain its basic relationship structure.

## Common mistakes to anticipate
- adding too many accent colors
- treating harmony rules as mandatory laws
- confusing visual interest with palette chaos

## Suggested feedback lines
- "This palette works because the colors feel related and the accent is controlled."
- "Try reducing the number of competing accents."
- "Harmony is about coordination, not maximum variety."

Reinforce the vocabulary and visual judgment skills from the unit before the milestone.

## Duration
10 to 15 minutes

## Review components
- term matching: hue, saturation, lightness, contrast, warm, cool, hierarchy, harmony
- rapid visual recognition drills
- fix-the-problem micro tasks
- one short reflection prompt asking the learner to describe color choices in a sample UI

## Recommended review mechanics
- spaced repetition cards
- 30-second visual decisions
- one "explain your choice" free response item
- optional retry path for weak areas

# Unit Milestone: Read the Interface

## Milestone purpose
Test whether the learner can use the unit's vocabulary to analyze color function in a simple interface.

## Duration
15 to 20 minutes

## Milestone setup
Show a simple but intentionally imperfect interface such as:
- a landing page hero
- a signup form
- a dashboard card set
- a settings screen

The design should include both strengths and weaknesses:
- one clear focal point
- one low-contrast issue
- one unnecessary color use
- one grouping or sectioning success
- one area where hierarchy could be improved

## Milestone tasks
The learner must:
1. identify the first thing the eye notices
2. explain which color creates that emphasis
3. identify one contrast problem
4. classify at least one color relationship or palette tendency
5. suggest one change to improve hierarchy
6. explain whether the palette feels mostly warm, cool, or balanced

## Submission/output format
The app can support one or more of the following:
- clickable annotations on the interface
- structured short answers
- drag-and-drop labels onto interface regions
- a final mini-redesign using guided controls

## Scoring rubric
### Strong performance
- uses terms correctly
- notices both readability and hierarchy issues
- suggests a practical color improvement
- explains the palette in functional terms rather than vague preference

### Partial performance
- identifies obvious color changes but struggles to name them precisely
- notices that something feels off but cannot explain why clearly

### Needs reinforcement
- relies on subjective language only, such as "pretty" or "ugly"
- misses contrast problems
- treats all color choices as equally important

## Milestone completion criteria
The learner passes when they can:
- correctly use at least 5 of the unit's core terms in context
- identify at least 1 real clarity problem
- suggest at least 1 believable improvement

## Badge unlocked
**Color Observer**

# Notes for implementation
- keep examples digital-first and interface-focused
- avoid fine-art framing unless it directly supports UI understanding
- use simple neutral UI mockups so color effects are easy to see
- avoid requiring perfect terminology on first exposure, but reinforce correct words immediately after learner input
- provide many comparison-based tasks because they are fast, game-like, and beginner-friendly
- make feedback specific to the visible change, not generic praise
- keep written responses short and structured so the app remains lightweight and approachable
